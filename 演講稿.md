## 二、核心技術：Antigravity 介紹與應用（約 8 分鐘）

### 什麼是 Antigravity？

Antigravity 是一個強大的 **程式碼自動生成工具**。它讓我們能夠用自然語言或簡潔的指令，快速生成網頁前端程式碼以及後端 API。

**過去的方式**：開發一套防災系統，需要前端、後端工程師耗費數週撰寫程式。  
**現在的方式**：用 Antigravity，幾小時甚至幾分鐘就能生成可用程式碼。

### 安裝與設定

```bash
# 以 Python 套件為例（實際工具請依官方文件）
pip install antigravity

# 或使用 Node.js
npm install -g antigravity
```

基本使用：

```bash
# 生成一個 Folium 災害地圖網頁
antigravity generate map --type=folium --output=index.html

# 生成 Gradio 災害通報 API
antigravity generate api --framework=gradio --endpoint=/report
```

Antigravity 會讀取簡單的設定檔（YAML 或 JSON），自動產出符合最佳實務的程式碼框架。

### 防災應用示例

**應用 1：即時雨量與土石流警戒看板**
- 用 Antigravity 生成一個 Folium 地圖網頁
- 顯示所有通報地點的風險等級（紅/橘/綠）

**應用 2：防災 API**
- 生成接收通報的 RESTful API
- 自動驗證輸入資料、計算風險等級、儲存到資料庫

## 三、無縫部署：GitHub Pages 與 Hugging Face Spaces（約 10 分鐘）

### 1. GitHub Pages - 靜態網頁代管

**原理**：GitHub Pages 是 GitHub 提供的免費靜態網頁代管服務，直接將倉庫中的 HTML/CSS/JS 檔案發佈為公開網站。

**部署步驟**：

1. 準備 `index.html` 和所有前端資源
2. 推送（push）到 GitHub 倉庫
3. 進入倉庫 Settings → Pages
4. Source 選擇 `main` 分支和 `/root`（或 `/docs` 資料夾）
5. 儲存後，網站即上線（URL: `https://username.github.io/repo-name/`）

**技術要點**：
- 自動 HTTPS 加密
- 支援自訂網域（CNAME 記錄）
- 每次 push 會自動重新部署

**局限**：
- 純靜態頁面，無法直接寫入資料庫
- 若需要持久化儲存，必須搭配後端 API（如 Hugging Face Spaces）或 GitHub Actions

**我們的應用**：將 Antigravity 生成的 `index.html` 部署，作為第一線人員的通報表單介面。

---

### 2. Hugging Face Spaces - 輕量級 API 與互動網頁

**原理**：Hugging Face Spaces 原本是 AI 模型展示平台，但它也支援 Gradio、Streamlit、Docker 等框架，非常適合部署輕量級後端服務或互動應用。

**部署步驟**：

1. 註冊 Hugging Face 帳號（huggingface.co）
2. 點選個人頭像 → **New Space**
3. 填寫：
   - Space name: `your-username/disaster-map`
   - License: MIT
   - SDK: **Gradio**
4. 將專案檔案上傳（或綁定 GitHub 倉庫自動同步）
   - 必須包含：`app.py`（主程式）、`requirements.txt`（相依清單）
5. Space 會自動安裝相依並啟動 Gradio 服務

**關鍵技術細節**：

我們的 `requirements.txt`：

```txt
gradio>=4.0
folium
pandas
```

我們的 `app.py`（核心功能）：
- 使用 Gradio 建立 Web 介面
- 使用 Folium 生成災害地圖
- 提供 API 介面供 Openclaw 呼叫
- 資料暫存於 `disaster_data.json`（或可連接資料庫）

**實際限制**：
- 免費 Space 有記憶體（約 16GB）和執行時間限制（適合輕量應用）
- 長時間任務可能超時，需優化或轉為非同步
- 若需 GPU 加速模型推論，需升級為 Pro 或 Enterprise

## 四、智慧助理：Openclaw 介紹與實作（約 8 分鐘）

網頁與 API 都上線了，那第一線巡查人員該如何快速回報與接收資訊呢？這就交給 **Openclaw**。

### 什麼是 Openclaw？

Openclaw 是一個**自動化工作流助手框架**，能夠整理資訊、串接多種通訊軟體，並執行自定義的業務邏輯。它就像一個 24 小時在線的數位助理。

### 核心功能

- **多平台通訊整合**：Telegram Bot、LINE Bot、Email (SMTP)、Webhook
- **工作流引擎**：用 YAML 或 JavaScript 定義自動化流程
- **外部 API 呼叫**：可呼叫 Hugging Face 模型、GitHub API、天氣資料等
- **條件判斷與通知**：根據災害等級自動發送 Email 或 LINE 提醒

### 應用場景

我們可以將 Openclaw 與大家最常用的 LINE 或 Telegram 綁定：

**情境**：水保署同仁在山区巡查發現崩塌跡象

1. 同仁用手機傳訊息給 Openclaw Bot：
   ```
   地點：XX縣XX路50號 near 山邊坡
   描述：看到大石頭裂開，有土石流風險
   （附照片一張）
   ```

2. Openclaw 收到訊息後自動執行：
   - **解析文字**：提取地點、描述
   - **呼叫 Hugging Face API**：判斷風險等級（高/中/低）
   - **更新災害資料**：推送新通報到 GitHub 的 `disaster_data.json`
   - **觸發通知**：發送 Email 給減災監測組群組
   - **回覆確認**：Bot 回傳「✅ 已記錄，風險等級：高，指揮中心已收到」

整個過程中，同仁只需傳一張照片和幾句話，所有後續處理都由 Openclaw 自動完成。

### 安裝與設定

```bash
# 安裝 Openclaw CLI
npm install -g openclaw   # 或 pip install openclaw

# 初始化專案
openclaw init disaster-bot

# 編輯設定檔
vim config.yaml
```

**設定檔範例** (`config.yaml`):

```yaml
telegram:
  bot_token: "YOUR_TELEGRAM_BOT_TOKEN"
  allowed_chats: []  # 留白則允許所有人，或填入允許的 chat ID

huggingface:
  api_url: "https://api-inference.huggingface.co/models/your-org/risk-classifier"
  token: "YOUR_HF_TOKEN"

github:
  repo: "your-username/disaster-dashboard"
  token: "YOUR_GITHUB_TOKEN"
  branch: "main"
  data_file: "disaster_data.json"

email:
  smtp_host: "smtp.gmail.com"
  smtp_port: 587
  username: "your-email@gmail.com"
  password: "YOUR_APP_PASSWORD"
  to_team: "emergency-team@yourdomain.com"

workflows:
  - name: "災害通報處理"
    trigger: "telegram.message"
    steps:
      - parse_message
      - call_huggingface
      - update_github
      - send_email_if_high_risk
      - reply_user
```

**詳細步驟**：
1. 在 Telegram 申請 Bot（@BotFather）取得 `bot_token`
2. 在 Hugging Face 建立模型 Space 或使用 Inference API 取得 `token`
3. 在 GitHub 建立 Personal Access Token（repo 權限）
4. 在 Email 服務商開啟 SMTP 並產生應用專用密碼
5. 將上述 Token 填入 `config.yaml`
6. 啟動 Openclaw：`openclaw start`

## 五、實戰情境模擬（綜合應用）（約 5 分鐘）

讓我們把這些工具串起來，想像一個真實的颱風應變場景：

### 整備期（颱風來臨前）

1. **水保署減災中心**利用 Antigravity 快速生成：
   - 一個針對本次颱風的「專屬災害監測網頁」（GitHub Pages）
   - 一個「雨量預警 API」（Hugging Face Spaces）

2. 部署完成：
   ```
   網頁：https://water-soil.gov.tw/disaster-dashboard
   API：https://cwbdayi-disaster.hf.space/api/analyze
   ```

### 應變期（颱風期間）

3. **各地區分局同仁**在現場勘查：
   - 用手機拍照、輸入地點和描述
   - 透過 LINE 傳送給 Openclaw Bot

4. **Openclaw 自動處理**：
   ```
   接收訊息 → 呼叫 Hugging Face 模型分析圖片 → 判斷等級為"高"
   → 更新 GitHub Pages 的 JSON 資料 → 地圖自動刷新
   → 發送 Email 給指揮中心：「XX縣有高等級通報！」
   → 回覆 LINE：「已處理，指揮中心已收到」
   ```

### 關鍵優勢

- **零 IT 依賴**：第一線人員直接回報，不需要填寫 Excel 再轉交 IT
- **即時同步**：網頁看板、API、Email 同時更新
- **低成本**：全部使用免費服務（GitHub、Hugging Face、Openclaw 自架）
- **可擴展**：可接入天氣 API、歷史資料分析、AI 影像辨識
