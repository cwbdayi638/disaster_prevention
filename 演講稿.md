## 二、核心技術：Antigravity 介紹與應用（約 8 分鐘）

### 什麼是 Antigravity？

Antigravity（反重力）代表的是** democratizing code development **的概念——讓非技術人員也能參與程式開發。在防災領域，這意味著：
- 第一線同仁可以用**描述性語言**定義需求（例如：「我需要一個能顯示雨量 Location 的地圖」）
- Antigravity 系統自動生成對應的程式碼框架
- 開發人员再微調或直接部署

在本演講的案例中，我們假設 Antigravity 是一個**多程式碼生成平台**，它支援：
- **前端生成**：HTML/CSS/JavaScript、React、Vue、Gradio 介面
- **後端生成**：Python Flask/FastAPI、Node.js Express、資料庫 schema
- **配置生成**：`requirements.txt`、`package.json`、`Dockerfile`、CI/CD 配置

### 實際應用：生成防災系統程式碼

**情境**：水保署需要一個新的土石流警戒看板

傳統流程：需求會議 → UI 設計 → 前端開發（1週） → 後端 API（1週） → 測試 → 部署（總計 2-4 週）

Antigravity 流程：
1. 用**自然語言描述**需求：
   ```
   請為我建立一個土石流防災地圖系統，需求如下：
   - 使用 Gradio 作為前端框架
   - 整合 Folium 地圖顯示所有通報位置
   - 新增通報表單（地點、描述、坐標、風險等級）
   - 風險等級用紅橘綠標示
   - 資料儲存為 JSON 格式
   ```

2. Antigravity 返回生成的檔案結構：
   ```
   disaster-map/
   ├── app.py              # Gradio 主程式
   ├── requirements.txt    # 依賴清單
   ├── static/
   │   └── style.css      # 自定樣式（如有）
   └── README.md          # 使用說明
   ```

3. 驗證並微調（如有需要）

### 技術細節：生成的程式碼長的樣子

Antigravity 最終生成的 `app.py`（我們實際專案的簡化版）：

```python
import gradio as gr
import folium
from folium.plugins import MarkerCluster
import json
import os
from datetime import datetime

DATA_FILE = "disaster_data.json"

# 初始示例資料
SAMPLE_DATA = [
    {
        "id": 1,
        "location": "XX縣XX路50號",
        "lat": 24.948199,
        "lng": 121.225912,
        "description": "山路邊坡出現裂痕，有土石滑落風險",
        "level": "中",
        "reported_at": "2025-03-10 00:00",
        "status": "待處理"
    }
]

def load_data():
    if os.path.exists(DATA_FILE):
        with open(DATA_FILE, "r", encoding="utf-8") as f:
            return json.load(f)
    else:
        save_data(SAMPLE_DATA)
        return SAMPLE_DATA

def save_data(data):
    with open(DATA_FILE, "w", encoding="utf-8") as f:
        json.dump(data, f, ensure_ascii=False, indent=2)

def create_map(data):
    """生成 Folium 地圖 HTML"""
    if data:
        center_lat = sum(d["lat"] for d in data) / len(data)
        center_lng = sum(d["lng"] for d in data) / len(data)
    else:
        center_lat, center_lng = 24.948199, 121.225912

    m = folium.Map(location=[center_lat, center_lng], zoom_start=12)
    marker_cluster = MarkerCluster().add_to(m)

    level_colors = {"高": "red", "中": "orange", "低": "green"}

    for item in data:
        color = level_colors.get(item["level"], "blue")
        popup_html = f"""
        <b>地點：</b> {item['location']}<br>
        <b>描述：</b> {item['description']}<br>
        <b>風險等級：</b> {item['level']}<br>
        <b>通報時間：</b> {item['reported_at']}<br>
        <b>狀態：</b> {item['status']}
        """
        folium.Marker(
            location=[item["lat"], item["lng"]],
            popup=folium.Popup(popup_html, max_width=300),
            tooltip=f"{item['location']} - {item['level']}風險",
            icon=folium.Icon(color=color, icon="warning", prefix="fa")
        ).add_to(marker_cluster)

    return m._repr_html_()

def get_report_list(data):
    """生成通報列表用於顯示"""
    if not data:
        return "暫無通報資料"
    lines = []
    for i, item in enumerate(data, 1):
        lines.append(f"{i}. [{item['level']}] {item['location']} - {item['description'][:30]}... ({item['reported_at']})")
    return "\n".join(lines)

def submit_report(location, description, lat, lng, level):
    """提交新通報"""
    if not location or not description:
        return "❌ 請填寫地點和描述", gr.update(), gr.update()

    try:
        lat = float(lat) if lat else 24.948199
        lng = float(lng) if lng else 121.225912
    except:
        lat, lng = 24.948199, 121.225912

    data = load_data()
    new_id = max([d["id"] for d in data], default=0) + 1
    new_item = {
        "id": new_id,
        "location": location,
        "lat": lat,
        "lng": lng,
        "description": description,
        "level": level,
        "reported_at": datetime.now().strftime("%Y-%m-%d %H:%M"),
        "status": "待處理"
    }
    data.append(new_item)
    save_data(data)

    return "✅ 通報已成功提交！", create_map(data), get_report_list(data)

# 初始加载数据
initial_data = load_data()

# 创建界面
with gr.Blocks(title="土石流防災即時資訊地圖") as demo:
    gr.Markdown("# ⚠️ 土石流防災即時資訊系統")
    gr.Markdown("整合 AI 防災助理，24小時自動監測與通報")

    with gr.Row():
        with gr.Column(scale=3):
            map_html = gr.HTML(value=create_map(initial_data))
        with gr.Column(scale=1):
            gr.Markdown("## 📋 通報列表")
            report_list = gr.Textbox(
                value=get_report_list(initial_data),
                label="",
                lines=20,
                interactive=False
            )

    with gr.Row():
        gr.Markdown("---")

    with gr.Row():
        with gr.Column():
            gr.Markdown("## 📝 提交新通報")
            with gr.Row():
                location_input = gr.Textbox(label="地點", placeholder="例如：XX縣XX路XX號附近")
                level_input = gr.Dropdown(choices=["低", "中", "高"], value="中", label="風險等級")
            description_input = gr.Textbox(label="詳細描述", placeholder="請描述具體情況（如山牆裂痕、土石滑動等）", lines=3)
            with gr.Row():
                lat_input = gr.Textbox(label="緯度", placeholder="可選，預設台北")
                lng_input = gr.Textbox(label="經度", placeholder="可選，預設台北")
            submit_btn = gr.Button("📤 送出通報", variant="primary")
            result_msg = gr.Textbox(label="提交結果", interactive=False)

    submit_btn.click(
        fn=submit_report,
        inputs=[location_input, description_input, lat_input, lng_input, level_input],
        outputs=[result_msg, map_html, report_list]
    )

if __name__ == "__main__":
    demo.launch()
```

## 三、無縫部署：GitHub Pages 與 Hugging Face Spaces（約 10 分鐘）

### 1. GitHub Pages - 靜態網頁代管的完整指南

GitHub Pages 適合托管**靜態內容**（HTML/CSS/JS），不需要伺服器。以下是詳細步驟：

#### 步驟 1：建立 GitHub 倉庫

```bash
# 如果已有專案，進入專案目錄
cd /path/to/disaster-frontend

# 初始化 git（如果還沒做）
git init
git add .
git commit -m "Initial commit: Antigravity generated frontend"

# 創建 GitHub 倉庫（假設已建立好空倉庫）
git remote add origin https://github.com/your-username/disaster-dashboard.git
git branch -M main
git push -u origin main
```

#### 步驟 2：啟用 GitHub Pages

1. 到 GitHub 倉庫頁面 → 點擊 **Settings**
2. 左側選單選擇 **Pages**
3. **Build and deployment** 選擇：
   - Source: `Deploy from a branch`
   - Branch: `main` → `/root`（或選擇 `/docs` 如果你把網頁放在 docs 資料夾）
4. 點擊 **Save**

等待約 30 秒，頁面會顯示：
```
Your site is live at https://your-username.github.io/disaster-dashboard/
```

#### 步驟 3（可選）：自訂網域

如果你有自有網域（如 `disaster.wra.gov.tw`）：

1. 在 `CNAME` 檔案中写入你的子域名
   ```
   disaster.wra.gov.tw
   ```
2. 在 DNS 管理員新增 CNAME 記錄：
   - **名稱/主機**：`disaster`（或 `www`）
   - **目標/指向**：`your-username.github.io`
3. 在 GitHub Pages 設定中勾選 **Enforce HTTPS**

#### 步驟 4：CI/CD 自動化（進階）

每次 Git push 後，GitHub Pages 會自動重建。如果你想在 push 前**自動測試**或**生成靜態檔案**，可以設定 GitHub Actions：

`.github/workflows/deploy.yml`：
```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v3

      - name: Setup Node.js (if using Node build tools)
        uses: actions/setup-node@v3
        with:
          node-version: '18'

      - name: Build static files (if needed)
        run: |
          # 如果需要构建步骤（例如 Vue/React）
          # npm run build

      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./  # 发布当前目录
```

#### 技術注意事項

- **索引文件**：GitHub Pages 預設尋找 `index.html` 或 `README.md`（Markdown 会自动转换成 HTML）
- **大小限制**：單個檔案不超过 100 MB，整個倉庫不超過 1 GB
- **HTTPS**：預設啟用，不支援自簽憑證
- **API 金鑰**：**絕對不要**將 API token 寫在靜態頁面中（會被公開）！所有機密資訊必須透過後端代理。

### 2. Hugging Face Spaces - 部署 Gradio API 的完整指南

Hugging Face Spaces 是部署 AI 應用和輕量 API 的最佳選擇，尤其適合 Gradio 框架。

#### 步驟 1：註冊與建立 Space

1. 前往 https://huggingface.co 註冊帳號
2. 點擊右上角頭像 → **New Space**
3. 填寫表單：
   - **Space name**: `your-username/disaster-map`（全小寫，用連字符）
   - **License**: MIT（或合适的开源协议）
   - **Space SDK**: **Gradio**
   - **Public/Private**: 公開（可被搜尋）或私有（只限自己）
4. 點擊 **Create Space**

#### 步驟 2：上傳程式碼

建立後，Space 頁面會顯示 Git 遠端 URL：
```
git remote add space https://huggingface.co/spaces/your-username/disaster-map
```

推送程式碼：
```bash
# 如果已經有本地端 git repo（我们的 Dayi 文件夹）
cd /home/ubuntu/.openclaw/workspace/Dayi

# 添加 Hugging Face 遠端
git remote add space https://huggingface.co/spaces/cwbdayi/disaster_prevention

# 推送（如果已有遠端，可跳過）
git push space main
```

** alternatively **：直接在 Hugging Face web 介面上傳檔案（適合少量檔案）

#### 步驟 3：確認部署成功

Space 頁面會顯示建置狀態：
- **Building** → 安裝相依、執行 `app.py`
- **Running** → 服務已啟動，顯示 URL：`https://cwbdayi-disaster.hf.space`
- **Failed** → 查看 **Logs** 除錯

#### 步驟 4：設定環境變數（Secrets）

如果程式碼需要 API token（例如 OpenAI、Hugging Face Inference API），請到 Space 設定：

1. 頂部選單 **Settings** → 左側 **Repository secrets**
2. 點擊 **New secret**
3. Key: `HF_TOKEN`（或自定義名稱）
4. Value: 你的 Hugging Face token（從 https://huggingface.co/settings/tokens 取得）
5. 勾選 **Include in builds** → **Add secret**

在你的 Python 程碼中讀取：
```python
import os
HF_TOKEN = os.getenv("HF_TOKEN")  # 安全，不會被公開
```

#### 步驟 5：使用自訂的 `requirements.txt`

你的 `requirements.txt` 非常重要，它決定建置環境：

```txt
# 核心框架
gradio>=4.0,<5.0  # Gradio 4.x 穩定版
folium>=0.16.0     # 地圖
pandas>=2.0.0      # 資料處理

# 如果有 ML 模型需求
torch>=2.0.0       # PyTorch（可選，會增大 build 時間）
transformers>=4.30.0  # Hugging Face 模型庫

# 如果需呼叫外部 API
requests>=2.30.0
aiohttp>=3.8.0     # 非同步 HTTP

# 其他
python-dotenv>=1.0.0  # 環境變數管理
```

**重要提示**：
- Hugging Face Spaces 免費版使用 **CPU**，GPU 需要付费（可申請研究贈送）
- `requirements.txt` 應指定版本，避免不相容
- 某些套件需系統相依（如 `libgl1-mesa-glx` 對 OpenCV），但 Gradio 通常不需要

#### 步驟 6：測試 API 端點

Gradio 自動提供 Web UI，但我們也可以**直接呼叫 API**：

 deployed 後，端點為：
```

```

使用 curl 測試：
```bash

  -H "Content-Type: application/json" \

      "XX縣XX路50號",
      "山路邊坡出現裂痕",
      "24.948199",
      "121.225912",
      "中"
    ]
  }'
```

#### 技術限制與最佳實踐

| 項目 | 免費版限制 | 建議 |
|------|-----------|------|
| 記憶體 | ~16 GB | 避免載入大模型 |
| CPU | 共享 | 長時間task用佇列 |
| 儲存 | 臨時檔案系統（重啟消失） | 使用 HF Dataset 或外部 DB |
| timeout | 推理幾分鐘 | 長時間用 background task |
| concurrent | 有限製 | 設定 queue 或 scaling |

**最佳實踐**：
1. 將靜態資源（CSS/JS）放在 `/static` 目錄，Gradio 會自動托管
2. 使用 `gr.Interface(..., cache_examples=True)` 快取結果
3. 大模型改用 Inference API（`requests.post("https://api-inference.huggingface.co/...")`）
4. 監控日誌：Space 頁面的 **Logs** 分頁

## 四、智慧助理：Openclaw 介紹與實作（約 8 分鐘）

### Openclaw 架構概觀

Openclaw（OpenClaw）是一個**可擴展的自動化工作流引擎**，核心特色：
- **跨平台通訊整合**：Telegram、LINE、Discord、Email、Slack、Webhook
- **事件驅動**：message → trigger → workflow → action
- **腳本化**：支援 YAML 配置、JavaScript/TypeScript、Python
- **插件架構**：可自定義 connector、transformer、notifier

本演講聚焦於 Openclaw 的 **Telegram Bot 整合案例**。

### Openclaw 與現有系統的連接方式

```
第一線同仁 (LINE/Telegram)
        ↓ 發送災害通報
Openclaw Bot (接收並解析)
        ↓ 提取地點、描述、圖片
[Hugging Face API]  → 評估風險等級（AI 模型）
        ↓ 生成結構化 JSON
[GitHub API]       → 更新 disaster_data.json
        ↓ 提交 PR 或直接 push
[GitHub Pages]     → 網頁地圖自動刷新（每次 reload）
        ↓
[Email/SLACK]      → 發送警報給減災組
        ↓
Openclaw Bot      → 回覆同仁「已處理」
```

**技術關鍵**：Openclaw 不直接改動 HTML 或 Gradio 程式碼，而是**更新數據檔案**（`disaster_data.json`）。Gradio 應用程式在每次 render 時**動態讀取最新 JSON**，因此無需重啟服務。

### 自定義 Handler（JavaScript 範例）

`handlers/github-utils.js`：
```javascript
// 輔助函數：更新 GitHub 的 JSON 檔案
function updateGithubFile({repo, branch, path, newItem, token}) {
  const { Octokit } = require("@octokit/rest");

  const octokit = new Octokit({ auth: token });

  // 1. 取得目前檔案內容
  const { data: file } = await octokit.repos.getContent({
    owner: repo.split("/")[0],
    repo: repo.split("/")[1],
    path: path,
    ref: branch
  });

  const content = Buffer.from(file.content, "base64").toString();
  const json = JSON.parse(content);

  // 2. 新增資料
  json.push(newItem);

  // 3. 計算新的 SHA（GitHub 要求）
  const newContent = Buffer.from(JSON.stringify(json, null, 2)).toString("base64");

  // 4. Commit & Push
  await octokit.repos.createOrUpdateFileContents({
    owner: repo.split("/")[0],
    repo: repo.split("/")[1],
    path: path,
    message: config.integrations.github.commit_message,
    content: newContent,
    sha: file.sha,
    branch: branch
  });

  return { success: true, updated_count: json.length };
}

module.exports = { updateGithubFile };
```

這個 handler 必須在 `workflows/` 的 `github.update_file` 動作中被引用。

### 取得 API Tokens（ practical 步驟）

#### 1. Telegram Bot Token

1. 打開 Telegram，搜尋 `@BotFather`
2. 發送 `/newbot`
3. 輸入 Bot 名稱（如 `Disaster Bot`）和用户名（如 `disaster_assistant_bot`）
4. BotFather 回傳：
   ```
   Done! Congratulations on your new bot. You will find it at t.me/disaster_assistant_bot.
   You can use this token to access the HTTP API:
   123456789:ABCdefGHIjklMNOpqrsTUVwxyz-1234567890
   ```
5. 將此 token 存入 `.env` 或環境變數 `TELEGRAM_BOT_TOKEN`

**設定群組權限**（可選）：
```bash
# 與 Bot 私聊開始
curl -X POST "https://api.telegram.org/bot<TOKEN>/getUpdates"
# 在群組中 @你的bot，讓它成為成員
```

#### 2. GitHub Personal Access Token

1. 登入 GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
2. Generate new token → Classic
3. Note: `Openclaw disaster bot`
4. Select scopes:
   - ✅ `repo`（完整 repo 控制權，包含 admin）
   - ✅ `workflow`（如果需要 GitHub Actions）
   - ❌ 其他不需要
5. Generate token → **複製並妥善保存**（只顯示一次）
6. 存入 `.env`：`GITHUB_TOKEN=ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`

#### 3. Hugging Face Token

1. 登入 Hugging Face → Settings → Access Tokens
2. Create new token
   - Role: `user`（或 `admin` 如果需管理 Space）
   - Expiration: 可選（建議 30 天）
3. 複製 token
4. 存入 `.env`：`HF_TOKEN=hf_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`

#### 4. Gmail App Password（如需 Email）

1. 登入 Google 帳戶 → Security
2. 開啟 **2-Step Verification**（必要）
3. 找到 **App passwords**
4. Select app: `Mail`，Select device: `Other (Custom name)` → `Openclaw Bot`
5. 產生 16 位密碼：`abcd efgh ijkl mnop`
6. 存入 `.env`：`EMAIL_PASSWORD=abcd...`

## 六、結語與 Q&A（約 6 分鐘）

### 總結：三大核心價值

1. **速度**：從需求到上線從數週縮短到幾小時
2. **門檻**：不需要龐大 IT 團隊，第一線人員可直接貢獻
3. **彈性**：所有工具都可單獨使用或混合搭配

### 技術棧總整理

```
需求 → Antigravity (生成程式碼)
      ↓
程式碼 → GitHub (版本控制)
      ↓
前端 → GitHub Pages (靜態托管)
後端 → Hugging Face Spaces (Gradio API)
整合 → Openclaw (工作流引擎)
使用者 → Telegram / LINE
```

### 常見問題預備

**Q1: 資料安全嗎？会不会被外人看到？**  
- GitHub Pages 預設是公開的，適合即時災情資訊（本意就是要公開讓民眾查看）
- 內部資料可用 Private repo，或用token限制 API 存取
- Openclaw 可設定 allowed_user_ids 只允許特定人員

**Q2: Hugging Face Spaces 有流量限制嗎？**  
- 免費版有每分钟请求数限制（约几十）
- 高流量需付费或用私有伺服器
- 但減災情境通常是一天幾十到幾百筆，免费足夠

**Q3: 如何確保坐標準確？**  
- 第一線人員可開啟手機 GPS 直接傳緯度
- 或使用 Google Maps API / OpenStreetMap Nominatim 將地址轉換為坐標（需額外 API）
- Openclaw 可整合 `geocoding` 服務自動轉換

**Q4: 如果 HF API 傳回錯誤或 timeout 怎麼辦？**  
- Openclaw 支援 retry 機制（`retry: 3`）
- 可設定 fallback 到 rule-based 評估（例如從關鍵字判斷）
- 所有失敗請求和錯誤都記錄到 `error.log`

**Q5: 系統如何備份與還原？**  
- GitHub本身就是版本控制，所有資料都在 repo 中
- 定期 export 為 CSV 或存入資料庫備份
- Hugging Face Spaces 有版本歷史，可 rollback

**Q6: 能否整合既有的政府系統（如災害管理系統）？**  
- 可以，Openclaw 支援 Webhook，可呼叫任何 REST API
- 或使用 GitHub 分支: 更新後自動觸發 GitHub Actions 同步到內部系統
