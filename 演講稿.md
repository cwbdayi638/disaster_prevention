## 二、核心技术：Antigravity 介绍与应用（约 8 分钟）

### 什么是 Antigravity？

Antigravity 是一个强大的 **代码自动生成工具**。它让我们能够用自然语言或简洁的指令，快速生成网页前端代码以及后端 API。

**过去的方式**：开发一套防灾系统，需要前端、后端工程师耗费数周撰写程式。  
**现在的方式**：用 Antigravity，几小时甚至几分钟就能生成可用代码。

### 安装与设定

```bash
# 以 Python 套件为例（实际工具请依官方文档）
pip install antigravity

# 或使用 Node.js
npm install -g antigravity
```

基本使用：

```bash
# 生成一个 Folium 灾害地图网页
antigravity generate map --type=folium --output=index.html

# 生成 Gradio 灾害通报 API
antigravity generate api --framework=gradio --endpoint=/report
```

Antigravity 会读取简单的配置文件（YAML 或 JSON），自动产出符合最佳实践的代码框架。

### 防灾应用示例

**应用 1：即时雨量与土石流警戒看板**
- 用 Antigravity 生成一个 Folium 地图网页
- 显示所有通报地点的风险等级（红/橙/绿）

**应用 2：防灾 API**
- 生成接收通报的 RESTful API
- 自动验证输入资料、计算风险等级、储存到数据库

## 三、无缝部署：GitHub Pages 与 Hugging Face Spaces（约 10 分钟）

### 1. GitHub Pages - 静态网页托管

**原理**：GitHub Pages 是 GitHub 提供的免费静态网页托管服务，直接将仓库中的 HTML/CSS/JS 文件发布为公开网站。

**部署步骤**：

1. 准备 `index.html` 和所有前端资源
2. 推送到 GitHub 仓库
3. 进入仓库 Settings → Pages
4. Source 选择 `main` 分支和 `/root` (或 `/docs` 文件夹)
5. 保存后，网站即上线（URL: `https://username.github.io/repo-name/`）

**技术要点**：
- 自动 HTTPS 加密
- 支持自定义域名（CNAME 记录）
- 每次 push 会自动重新部署

**局限**：
- 纯静态页面，无法直接写入数据库
- 若需要持久化存储，必须搭配后端 API（如 Hugging Face Spaces）或 GitHub Actions

**我们的应用**：将 Antigravity 生成的 `index.html` 部署，作为第一线人员的通报表单界面。

---

### 2. Hugging Face Spaces - 轻量级 API 与互动网页

**原理**：Hugging Face Spaces 原本是 AI 模型展示平台，但它也支持 Gradio、Streamlit、Docker 等框架，非常适合部署轻量级后端服务或互动应用。

**部署步骤**：

1. 注册 Hugging Face 账号（huggingface.co）
2. 点击个人头像 → **New Space**
3. 填写：
   - Space name: `your-username/disaster-map`
   - License: MIT
   - SDK: **Gradio**
4. 将项目文件上传（或绑定 GitHub 仓库自动同步）
   - 必须包含：`app.py`（主程式）、`requirements.txt`（依赖清单）
5. Space 会自动安装依赖并启动 Gradio 服务

**关键技术细节**：

我们的 `requirements.txt`：

```txt
gradio>=4.0
folium
pandas
```

我们的 `app.py`（核心功能）：
- 使用 Gradio 建立 Web 界面
- 使用 Folium 生成灾害地图
- 提供 API 接口供 Openclaw 调用
- 数据暂存于 `disaster_data.json`（或可连接数据库）

**实际限制**：
- 免费 Space 有内存（约 16GB）和执行时间限制（适合轻量应用）
- 长时间任务可能超时，需优化或转为异步
- 若需 GPU 加速模型推理，需升级为 Pro 或 Enterprise

## 四、智慧助理：Openclaw 介绍与实作（约 8 分钟）

网页与 API 都上线了，那第一线巡查人员该如何快速回报与接收资讯呢？这就交给 **Openclaw**。

### 什么是 Openclaw？

Openclaw 是一个**自动化工作流助手框架**，能够整理资讯、串接多种通讯软件，并执行自定义的业务逻辑。它就像一个 24 小时在线的数字助理。

### 核心功能

- **多平台通讯整合**：Telegram Bot、LINE Bot、Email (SMTP)、Webhook
- **工作流引擎**：用 YAML 或 JavaScript 定义自动化流程
- **外部 API 调用**：可呼叫 Hugging Face 模型、GitHub API、天气资料等
- **条件判断与通知**：根据灾害等级自动发送 Email 或 LINE 提醒

### 应用场景

我们可以将 Openclaw 与大家最常用的 LINE 或 Telegram 绑定：

**情境**：水保署同仁在山区巡查发现崩塌迹象

1. 同仁用手机传讯息给 Openclaw Bot：
   ```
   地点：XX县XX路50号 near 山边坡
   描述：看到大石头裂开，有土石流风险
   （附照片一张）
   ```

2. Openclaw 收到讯息后自动执行：
   - **解析文字**：提取地点、描述
   - **调用 Hugging Face API**：判断风险等级（高/中/低）
   - **更新灾害资料**：推送新通报到 GitHub 的 `disaster_data.json`
   - **触发通知**：发送 Email 给减灾监测组群组
   - **回复确认**：Bot 回传 "✅ 已记录，风险等级：高，指挥中心已收到"

整个过程中，同仁只需传一张照片和几句话，所有后续处理都由 Openclaw 自动完成。

### 安装与设定

```bash
# 安装 Openclaw CLI
npm install -g openclaw   # 或 pip install openclaw

# 初始化专案
openclaw init disaster-bot

# 编辑配置文件
vim config.yaml
```

**配置文件示例** (`config.yaml`):

```yaml
telegram:
  bot_token: "YOUR_TELEGRAM_BOT_TOKEN"
  allowed_chats: []  # 留空则允许所有人，或填入允许的 chat ID

huggingface:
  api_url: "https://api-inference.huggingface.co/models/your-org/risk-classifier"
  token: "YOUR_HF_TOKEN"

github:
  repo: "your-username/disaster-dashboard"
  token: "YOUR_GITHUB_TOKEN"
  branch: "main"
  data_file: "disaster_data.json"

email:
  smtp_host: "smtp.gmail.com"
  smtp_port: 587
  username: "your-email@gmail.com"
  password: "YOUR_APP_PASSWORD"
  to_team: "emergency-team@yourdomain.com"

workflows:
  - name: "灾害通报处理"
    trigger: "telegram.message"
    steps:
      - parse_message
      - call_huggingface
      - update_github
      - send_email_if_high_risk
      - reply_user
```

**详细步骤**：
1. 在 Telegram 申请 Bot（@BotFather）取得 `bot_token`
2. 在 Hugging Face 建立模型 Space 或使用 Inference API 取得 `token`
3. 在 GitHub 建立 Personal Access Token（repo 权限）
4. 在 Email 服务商开启 SMTP 并生成应用专用密码
5. 将上述 Token 填入 `config.yaml`
6. 启动 Openclaw：`openclaw start`

## 五、实战情境模拟（综合应用）（约 5 分钟）

让我们把这些工具串起来，想像一个真实的颱风应變场景：

### 整备期（台风来临前）

1. **水保署减災中心**利用 Antigravity 快速生成：
   - 一个针对本次颱风的「专属灾害监测网页」（GitHub Pages）
   - 一个「雨量预警 API」（Hugging Face Spaces）

2. 部署完成：
   ```
   网页：https://water-soil.gov.tw/disaster-dashboard
   API：https://cwbdayi-disaster.hf.space/api/analyze
   ```

### 应变期（台风期间）

3. **各地区分局同仁**在现场勘查：
   - 用手机拍照、输入地点和描述
   - 透过 LINE 传送给 Openclaw Bot

4. **Openclaw 自动处理**：
   ```
   接收讯息 → 调用 Hugging Face 模型分析图片 → 判断等级为"高"
   → 更新 GitHub Pages 的 JSON 数据 → 地图自动刷新
   → 发送 Email 给指挥中心："XX县有高等级通报！"
   → 回复 LINE："已处理，指挥中心已收到"
   ```

### 关键优势

- **零 IT 依赖**：第一线人员直接回报，不需要填写 Excel 再转交 IT
- **即时同步**：网页看板、API、Email 同时更新
- **低成本**：全部使用免费服务（GitHub、Hugging Face、Openclaw 自架）
- **可扩展**：可接入天气 API、历史资料分析、AI 影像识别
